### 2026-02-07 타입 export 작업

- 고려안:
  1. 각 스키마 클래스를 내부 경로에서 직접 export한다.
  2. handler 함수로 스키마 생성자를 래핑하고 `index.ts`에서 public API로 export한다.
- 선택: 2
- 이유:
  1. 외부 사용자에게 노출되는 경로를 단일화해 사용성을 단순화한다.
  2. `Infer` 타입과 함께 object 스키마의 제네릭 추론을 일관되게 제공한다.
- 아쉬운 점:
  1. 클래스 직접 사용이 필요한 경우 내부 경로가 감춰져 유연성이 감소될 수 있다.

### 2026-02-01 기본 베이스용 추상클래스 적용

- 고려안:
  1. 스키마 인터페이스만 상속받는다.
  2. 스키마 인터페이스를 상속받는 공통기능을 제공하는 추상 클래스를 만든다
- 선택: 2
- 이유:
  1. 기본 타입에 한해서 safeParse는 같은 기능을 가지고 수정이 필요할때 하나의 메소드만 수정하면 문제가 해결된다.
  2. 복잡한 타입의 경우에는 safeParse를 오버라이딩해서 쓰도록 구현할 수 있다.
- 아쉬운 점:
  1. 기본 타입들의 에러 메세지 전달 개선이 필요하다
